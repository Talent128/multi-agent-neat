阶段一：
第 0 步：原NEAT实现进化 N 代
初始化 NEAT 种群  ->  个体与环境交互  ->  根据 fitness 做进化  ->  持续运行 n 代  ->  选择最优个体固定拓扑
为什么要固定拓扑？
1. 后续会引入 进化强化协同优化，而RL 更擅长优化固定可微结构上的参数，而不是一边训练一边改结构。
2. 不能做参数软更新：如果两个网络拓扑不同，它们的参数无法一一对应，就不能做类似：$$\theta \leftarrow \tau \theta_{elite} + (1-\tau)\theta_{rl}$$
3. 另一种方案不用固定拓扑：EA-Master：EA 是主循环，RL 不再维护独立 policy，只保留一个 Critic；训练时用种群中一半个体的数据去训练 Critic，再用 Critic 给这部分个体做 policy-gradient 式引导，更新后的个体重新放回种群，然后整个人口再按 fitness 继续进化。但这个做法 时间复杂度高，且在参考文献中也有实验说明该方法最差。
4. 这一步之后，后续 EA 与 RL 才真正能在同一个参数空间里协同工作
这一步的作用可以前期快速搜索可行解，也可作为后续RL侧的 actor 暖启动。
阶段二：
EA + RL双向耦合，当奖励可靠时，让 RL 更深地参与进化；当奖励不可靠时，让 EA 保持更大的独立探索权：
EA 负责广泛探索 → 经验喂给 RL → RL 用 critic 做高效更新 → RL 策略回注种群 → elite 再反向软更新 RL
第 1 步：同时维护两个搜索系统
一边是 RL 分支：包含 actor 和 critic。
 另一边是 EA 分支：维护一个策略种群。
 总框架是 Parallel mode（并行模式），也就是 EA 和 RL 同时搜索策略空间，而不是谁完全主导谁。
第 2 步：EA 分支做“种群探索”
种群中的每个个体与环境交互，得到回报，作为适应度。
因为后续只做参数更新：EA侧可继续用neat（遗传算法），也可考虑用CEM（Cross-Entropy Method：维护一个搜索分布，并不断让这个分布朝优秀区域收缩）
参考文献中实验后说发现 CEM（Cross-Entropy Method） 比 Vanilla GA、PD-GA、Behavior-level GA 更稳、更强，尤其在大多数任务上更有优势，因为它不是盲目做参数级交叉变异，而是维护一个高斯分布，用优秀个体去更新均值和协方差，从而更高效地引导搜索。
第 3 步：EA 产生的轨迹喂给 RL
这是第一层融合，也是最经典的一层：
EA 在评估种群时产生大量多样化经验，这些经验进入共享 replay buffer，RL 用它来训练 critic/actor。即👉 replay buffer 里的数据不仅来自 RL 自己，还来自 EA population
这一步的意义是：EA 天然更擅长全局探索，能补足 RL 容易陷入局部最优、探索不足的问题。
可优化方向：精英交互：因为若所有个体都与环境进行交互，以获取适应度（累积奖励）并将所有生成的样本存入重放缓冲区。这种方式可能会导致低适应度个体产生低质量的样本进入缓冲区，影响RL优化效果。
第 4 步：RL 再把学到的策略反向注入种群
RL 在 replay buffer 上更新出一个更强的策略后，这个 RL policy 会被注入种群，参与下一轮进化竞争。
 如果它足够好，就会成为 elite；不够好，就会被淘汰。
这就是第二层融合：RL 的梯度优化结果回流到 EA，让种群不只是靠随机扰动前进，而是持续吸收 critic 提供的局部精修能力。
第 5 步：再加入“EA 直接改 RL”的通道
仅靠“经验共享 + 策略注入”还不够，参考文献中 比较了两种“EA 直接影响 RL”的机制：
-  Elite Guide 
-  Genetic Soft Update 
结果是 Genetic Soft Update 更有效。它直接把精英个体参数软更新到 RL 策略上：
$$\theta \leftarrow \tau \theta_{elite} + (1-\tau)\theta_{rl}$$
直观上，这相当于把 EA 找到的“好方向”直接灌进 RL，增强 RL 策略的可塑性，让 RL 更容易跳出坏局部最优。作者也明确说，这种改动能让 RL 更充分利用已有经验。

总结：经验层融合：EA → RL   参数层融合：EA → RL   策略层融合：RL → EA
