## 训练流程总览

### 🔄 NEAT训练循环

```
开始
  ↓
创建初始种群（随机基因组）
  ↓
┌─────────────────────────────┐
│  世代循环 (Generation Loop)  │
│                             │
│  1. 基因组 → 网络转换        │
│  2. 评估适应度              │
│  3. 选择父代                │
│  4. 交叉变异                │
│  5. 物种划分                │
└─────────────────────────────┘
  ↓
达到终止条件？
  ↓
返回最佳基因组
```

### 🎯 三种网络的训练流程对比

| 阶段 | RecurrentNet | AdaptiveLinearNet | AdaptiveNet |
|------|-------------|-------------------|-------------|
| **基因组表示** | 直接编码权重 | 编码CPPN | 编码6个CPPN |
| **网络创建** | 提取权重 → 矩阵 | 基因组 → CPPN → 初始权重 | 基因组 → 6个CPPN → 多个权重矩阵 |
| **前向传播** | 标准RNN | 线性层 + 权重更新 | RNN + 权重更新 |
| **评估** | 运行episode获得奖励 | 运行episode获得奖励 | 运行episode获得奖励 |
| **进化对象** | 权重值 | CPPN的拓扑和权重 | 6个CPPN的共享拓扑和权重 |

## 基因组详解

### 📦 NEAT基因组结构

NEAT基因组包含两部分：

```python
genome = {
    'nodes': {
        node_id: {
            'bias': float,        # 偏置
            'response': float,    # 响应系数
            'activation': str,    # 激活函数名
            'aggregation': str    # 聚合函数名
        }
    },
    'connections': {
        (in_id, out_id): {
            'weight': float,      # 连接权重
            'enabled': bool       # 是否激活
        }
    }
}
```

### 🔀 直接编码 vs CPPN编码

#### 1️⃣ RecurrentNet - 直接编码

**基因组 → 网络的转换**：

```python
# 步骤1：基因组定义网络拓扑
genome.nodes = {
    -1: input1,
    -2: input2,
    0: hidden1,
    1: output1
}

genome.connections = {
    (-1, 0): weight=0.5,   # input1 → hidden1
    (-2, 0): weight=-0.3,  # input2 → hidden1
    (0, 1): weight=1.2,    # hidden1 → output1
}

# 步骤2：提取权重填充矩阵
W_ih = [[0.5, -0.3]]  # 输入→隐藏
W_ho = [[1.2]]        # 隐藏→输出

# 步骤3：创建RecurrentNet
net = RecurrentNet(
    input_to_hidden=W_ih,
    hidden_to_output=W_ho,
    ...
)
```

**特点**：
- ✅ 简单直接，每个权重是一个基因
- ❌ 参数量大，搜索空间大
- ✅ 行为可预测

#### 2️⃣ AdaptiveLinearNet - CPPN间接编码

**基因组 → CPPN → 网络权重的转换**：

```python
# 步骤1：基因组定义CPPN网络
genome.nodes = {
    -1: x_in_leaf,
    -2: y_in_leaf,
    -3: x_out_leaf,
    -4: y_out_leaf,
    -5: pre_leaf,
    -6: post_leaf,
    -7: w_leaf,
    0: hidden_node,
    1: delta_w_output  # CPPN的输出
}

genome.connections = {
    (-1, 0): weight=0.8,   # x_in → hidden
    (-3, 0): weight=-0.5,  # x_out → hidden
    (0, 1): weight=1.0,    # hidden → delta_w
}

# 步骤2：基因组 → CPPN
cppn = create_cppn(
    genome, config,
    leaf_names=['x_in', 'y_in', 'x_out', 'y_out', 'pre', 'post', 'w'],
    node_names=['delta_w']
)

# 步骤3：使用CPPN生成初始权重矩阵
for i, out_coord in enumerate(output_coords):
    for j, in_coord in enumerate(input_coords):
        # 为每个连接调用CPPN
        weight[i, j] = cppn(
            x_in=in_coord[0],
            y_in=in_coord[1],
            x_out=out_coord[0],
            y_out=out_coord[1],
            pre=0, post=0, w=0
        )

# 步骤4：创建AdaptiveLinearNet
net = AdaptiveLinearNet(
    delta_w_node=cppn,  # 保存CPPN用于权重更新
    input_coords=input_coords,
    output_coords=output_coords,
    ...
)
```

**特点**：
- ✅ 参数效率高（CPPN小，生成权重多）
- ✅ 利用空间规律性
- ❌ 需要设计坐标
- ✅ 可以实现对称等几何特性

#### 3️⃣ AdaptiveNet - 多CPPN编码

**一个基因组编码6个CPPN**：

```python
# 步骤1：基因组定义一个大CPPN网络
# 有7个输入节点，6个输出节点
genome.output_keys = [1, 2, 3, 4, 5, 6]  # 6个输出

# 步骤2：分离成6个CPPN
nodes = create_cppn(
    genome, config,
    leaf_names=['x_in', 'y_in', 'x_out', 'y_out', 'pre', 'post', 'w'],
    node_names=['w_ih', 'b_h', 'w_hh', 'b_o', 'w_ho', 'delta_w']
)

cppn_w_ih = nodes[0]     # 生成输入→隐藏权重
cppn_b_h = nodes[1]      # 生成隐藏层偏置
cppn_w_hh = nodes[2]     # 生成隐藏→隐藏初始权重
cppn_b_o = nodes[3]      # 生成输出层偏置
cppn_w_ho = nodes[4]     # 生成隐藏→输出权重
cppn_delta_w = nodes[5]  # 生成权重更新规则

# 步骤3：使用6个CPPN生成所有权重矩阵
# （过程类似AdaptiveLinearNet，但有6个权重矩阵）

# 步骤4：创建AdaptiveNet
net = AdaptiveNet(
    w_ih_node=cppn_w_ih,
    b_h_node=cppn_b_h,
    # ... 其他CPPN
    delta_w_node=cppn_delta_w,
    ...
)
```

**特点**：
- ✅ 最强大的表达能力
- ✅ 6个CPPN共享基因组，协同进化
- ❌ 最复杂
- ❌ 训练最慢

## CPPN完整解析

### 🧠 CPPN是什么？

**CPPN (Compositional Pattern Producing Networks) = 组合模式生成网络**

CPPN是一个**特殊用途的神经网络**，它不是用来处理任务的，而是用来**生成权重**的。

#### 🔍 关键理解

```
普通神经网络：
输入（环境状态）→ 神经网络 → 输出（动作）

CPPN：
输入（坐标、激活值）→ CPPN → 输出（权重值或权重更新量）
```

### 📊 CPPN的输入输出

#### CPPN的7个输入

```python
cppn_inputs = {
    'x_in': 输入神经元的x坐标,    # 例如: -1.0
    'y_in': 输入神经元的y坐标,    # 例如: 0.0
    'x_out': 输出神经元的x坐标,   # 例如: 1.0
    'y_out': 输出神经元的y坐标,   # 例如: 1.0
    'pre': 输入神经元的激活值,    # 例如: 0.8（前向传播时）
    'post': 输出神经元的激活值,   # 例如: 0.3（前向传播时）
    'w': 当前的连接权重值          # 例如: 0.5（前向传播时）
}
```

#### CPPN的输出

**AdaptiveLinearNet**: 1个输出
```python
cppn_output = delta_w  # 权重更新量
```

**AdaptiveNet**: 6个输出
```python
cppn_outputs = {
    'w_ih': 输入→隐藏的权重,
    'b_h': 隐藏层偏置,
    'w_hh': 隐藏→隐藏的初始权重,
    'b_o': 输出层偏置,
    'w_ho': 隐藏→输出的权重,
    'delta_w': 隐藏→隐藏的权重更新量
}
```

### 🔄 CPPN的完整工作流程

#### 阶段1：初始化（使用CPPN生成初始权重）

```python
# AdaptiveLinearNet的初始化
net = AdaptiveLinearNet.create(genome, config, input_coords, output_coords)

# 内部流程：
def reset(self):
    # 1. 为每对(输入神经元, 输出神经元)调用CPPN
    for i, out_coord in enumerate(output_coords):
        for j, in_coord in enumerate(input_coords):
            # 2. 准备CPPN输入（初始化时激活值都是0）
            x_in = in_coord[0]      # 例如: -1.0
            y_in = in_coord[1]      # 例如: 0.0
            x_out = out_coord[0]    # 例如: 1.0
            y_out = out_coord[1]    # 例如: 1.0
            pre = 0.0               # 初始化时为0
            post = 0.0              # 初始化时为0
            w = 0.0                 # 初始化时为0
            
            # 3. 调用CPPN生成初始权重
            initial_weight = cppn(
                x_in=x_in, y_in=y_in,
                x_out=x_out, y_out=y_out,
                pre=pre, post=post, w=w
            )
            
            # 4. 保存到权重矩阵
            W[i, j] = initial_weight
    
    # 5. 应用阈值化（稀疏化）
    W[abs(W) < threshold] = 0
```

#### 阶段2：前向传播（使用CPPN更新权重）

```python
# 每个时间步的前向传播
def activate(self, inputs):
    # 1. 使用当前权重计算输出
    outputs = activation(W @ inputs)
    
    # 2. 准备CPPN输入（现在有真实的激活值）
    for i, out_coord in enumerate(output_coords):
        for j, in_coord in enumerate(input_coords):
            x_in = in_coord[0]
            y_in = in_coord[1]
            x_out = out_coord[0]
            y_out = out_coord[1]
            pre = inputs[j]         # 输入神经元的激活值
            post = outputs[i]       # 输出神经元的激活值
            w = W[i, j]             # 当前权重值
            
            # 3. 调用CPPN计算权重更新量
            delta_w = cppn(
                x_in=x_in, y_in=y_in,
                x_out=x_out, y_out=y_out,
                pre=pre, post=post, w=w
            )
            
            # 4. 更新权重（仅在表达的连接上）
            if W_expressed[i, j]:
                W[i, j] += delta_w
    
    # 5. 限制权重在合理范围
    W = clip(W, -weight_max, weight_max)
    
    return outputs
```

### 💡 CPPN的设计直觉

#### 为什么使用坐标？

```
坐标编码了神经元之间的空间关系！

例如在T-Maze中：
- 左传感器 [-1, 0] ↔ 左动作 [-1, 1]：距离近，可能权重大
- 右传感器 [1, 0] ↔ 左动作 [-1, 1]：距离远，可能权重小
```

CPPN可以学会利用这种几何规律性：
- **对称性**：左右对称的坐标产生相似的权重
- **距离依赖**：距离近的神经元连接更强
- **局部性**：相邻的神经元相互影响

#### 为什么使用激活值？

```
激活值让权重更新具有"学习"能力！

Hebbian学习规则: Δw = η × pre × post
- 如果输入和输出同时激活 → 增强连接
- 如果不同时激活 → 削弱连接
```

CPPN可以实现更复杂的规则：
```
delta_w = CPPN(坐标, pre, post, w)
```

这比固定的Hebbian规则更灵活，可以学到任务特定的更新规则。

## 坐标设计原理

### 🎨 为什么需要坐标？

坐标是**空间编码**的一种方式，用于告诉CPPN神经元之间的几何关系。

#### 没有坐标（RecurrentNet）
```
神经元只有ID: input1, input2, hidden1, output1
NEAT必须进化每一个连接的权重
```

#### 有坐标（AdaptiveLinearNet/AdaptiveNet）
```
神经元有位置: input1[-1,0], input2[0,0], output1[0,1]
CPPN可以利用位置关系生成权重
参数效率更高
```

### 📐 坐标设计实例详解

#### 示例：T-Maze任务

```python
# T-Maze环境的观测
observation = [
    wall_left,    # 左侧是否有墙（0或1）
    wall_front,   # 前方是否有墙（0或1）
    wall_right,   # 右侧是否有墙（0或1）
    color         # 颜色提示（奖励在左还是右）
]

# 动作
actions = [
    turn_left,    # 左转
    go_forward,   # 前进
    turn_right    # 右转
]
```

#### 坐标设计

```python
input_coords = [
    [-1.0, 0.0],  # 输入1：左侧墙壁传感器
    [0.0, 0.0],   # 输入2：前方墙壁传感器
    [1.0, 0.0],   # 输入3：右侧墙壁传感器
    [0.0, -1.0]   # 输入4：颜色传感器（特殊位置）
]

output_coords = [
    [-1.0, 0.0],  # 输出1：左转动作
    [0.0, 0.0],   # 输出2：前进动作
    [1.0, 0.0]    # 输出3：右转动作
]
```

#### 设计原理

```
1️⃣ 水平排列传感器（y=0）
   [-1, 0]        [0, 0]        [1, 0]
   左传感器       前传感器      右传感器
   
   → 反映了物理传感器的空间排列

2️⃣ 颜色传感器特殊位置（y=-1）
   [0, -1]
   颜色传感器
   
   → 不同于墙壁传感器，y坐标不同表示语义不同

3️⃣ 水平排列动作（y=0，与传感器同层）
   [-1, 0]        [0, 0]        [1, 0]
   左转           前进          右转
   
   → 与传感器对齐，利用几何对应

4️⃣ 期望的CPPN行为
   左传感器[-1,0] → 左转[-1,0]: 距离=0，可能强连接
   左传感器[-1,0] → 右转[1,0]:  距离=2，可能弱连接
   右传感器[1,0]  → 右转[1,0]:  距离=0，可能强连接
   
   颜色[0,-1] → 左转[-1,0]: 需要学习（记忆任务）
   颜色[0,-1] → 右转[1,0]:  需要学习（记忆任务）
```

### 🎯 坐标设计指南

#### 原则1：反映语义关系

```python
# ✅ 好的设计：传感器按方向排列
input_coords = [
    [-1, 0],  # 左
    [0, 0],   # 中
    [1, 0]    # 右
]

# ❌ 不好的设计：随机排列
input_coords = [
    [0.3, -0.7],  # 没有明确语义
    [-0.2, 0.5],
    [0.8, 0.1]
]
```

#### 原则2：利用对称性

```python
# 如果任务有对称性，坐标也应该对称
# 例如，左右对称的任务
input_coords = [
    [-1, 0],  # 左传感器
    [0, 0],   # 中间传感器
    [1, 0]    # 右传感器（与左传感器对称）
]

output_coords = [
    [-1, 0],  # 左动作
    [1, 0]    # 右动作（对称）
]
```

#### 原则3：使用不同的y坐标区分类别

```python
# 区分不同类型的输入
input_coords = [
    [-1, 0], [0, 0], [1, 0],  # 墙壁传感器（y=0）
    [0, -1]                    # 颜色传感器（y=-1，不同类型）
]

# 或者使用多层结构
input_coords = [
    [-1, 0], [0, 0], [1, 0],   # 第一层：空间输入
    [-1, 1], [0, 1], [1, 1]    # 第二层：时间输入
]
```

#### 原则4：范围通常在[-2, 2]

```python
# ✅ 好的范围
coords = [[-1, 0], [0, 0], [1, 0]]  # 在[-1, 1]范围内

# ⚠️ 可以但不常见
coords = [[-2, -2], [0, 0], [2, 2]]  # 更大范围

# ❌ 避免过大的值
coords = [[-10, 0], [0, 0], [10, 0]]  # 可能导致CPPN行为不稳定
```

### 🔬 实验：坐标设计的影响

```python
# 实验1：紧凑排列 vs 稀疏排列
design1 = [[-0.5, 0], [0, 0], [0.5, 0]]   # 紧凑
design2 = [[-2, 0], [0, 0], [2, 0]]       # 稀疏

# 结果：紧凑排列通常更好（神经元间距小，容易建立连接）

# 实验2：对称 vs 不对称
design1 = [[-1, 0], [0, 0], [1, 0]]       # 对称
design2 = [[-1, 0], [0.3, 0], [0.8, 0]]   # 不对称

# 结果：对称设计在对称任务上性能更好

# 实验3：语义对齐 vs 随机
design1 = [  # 语义对齐
    [-1, 0],  # 左传感器
    [-1, 1]   # 左动作（x坐标对齐）
]
design2 = [  # 随机
    [-1, 0],  # 左传感器
    [1, 1]    # 右动作（不对齐）
]

# 结果：语义对齐更容易学习
```

### 📤 网络输出设计

#### 1. 确定动作维度

```python
# 离散动作空间
env.action_space  # Discrete(2) → 2个动作

# 连续动作空间
env.action_space  # Box(-1, 1, (2,)) → 2维连续动作
```

#### 2. 设计输出坐标

```python
# 离散动作（如左/右）
output_coords = [
    [-1, 0],    # 动作0：左
    [1, 0]      # 动作1：右
]

# 多个离散动作

#### 3. 动作转换函数

# 对于RecurrentNet和自适应网络，需要转换输出为动作

# 示例1：二分类（CartPole）
def activate_net(net, states):
    outputs = net.activate(states)  # (batch_size, 1)
    actions = (outputs[:, 0] > 0.5).astype(int)  # 阈值0.5
    return actions

# 示例2：多分类（T-Maze）
def activate_net(net, states):
    outputs = net.activate(states)  # (batch_size, 3)
    actions = outputs.argmax(axis=1)  # 选择最大值的索引
    return actions

# 示例3：连续动作
def activate_net(net, states):
    outputs = net.activate(states)  # (batch_size, 2)
    actions = np.tanh(outputs)  # 限制在[-1, 1]
    return actions
```

### 🎯 完整示例：T-Maze

```python
# 1. 环境观测
"""
T-Maze观测空间：
- obs[0]: 左侧有墙？ (0或1)
- obs[1]: 前方有墙？ (0或1)
- obs[2]: 右侧有墙？ (0或1)
- obs[3]: 颜色提示 (0-1之间的值)
"""

# 2. 输入坐标设计
input_coords = [
    [-1.0, 0.0],   # obs[0] 左墙：放在左侧
    [0.0, 0.0],    # obs[1] 前墙：放在中间
    [1.0, 0.0],    # obs[2] 右墙：放在右侧
    [0.0, -1.0]    # obs[3] 颜色：特殊位置（不同y坐标）
]

# 设计理由：
# - 前三个按空间位置排列（左-中-右）
# - 颜色传感器在不同y坐标，表示它是不同类型的输入
# - y=0表示墙壁传感器，y=-1表示颜色传感器

# 3. 动作空间
"""
T-Maze动作空间：
- action=0: 左转
- action=1: 前进
- action=2: 右转
"""

# 4. 输出坐标设计
output_coords = [
    [-1.0, 0.0],   # action=0 左转：放在左侧
    [0.0, 0.0],    # action=1 前进：放在中间
    [1.0, 0.0]     # action=2 右转：放在右侧
]

# 设计理由：
# - 与输入传感器在同一y坐标（y=0）
# - 空间对齐：左传感器[-1,0] 对应 左转[-1,0]
# - 期望CPPN学到：左传感器有墙 → 不要左转

# 5. 创建网络
net = AdaptiveLinearNet.create(
    genome, config,
    input_coords=input_coords,
    output_coords=output_coords,
    batch_size=4
)

# 6. 动作选择函数
def activate_net(net, states):
    """
    states: (batch_size, 4) 的numpy数组
    returns: (batch_size,) 的整数数组，值为0/1/2
    """
    outputs = net.activate(states)  # (batch_size, 3)
    actions = outputs.argmax(axis=1)  # 选最大的
    return actions.numpy()

# 7. 评估循环
for step in range(max_steps):
    # 前向传播
    actions = activate_net(net, states)
    
    # 环境交互
    next_states, rewards, dones, truncated, infos = env.step(actions)
    
    # 累积奖励
    total_rewards += rewards
    
    # 更新状态
    states = next_states
```

## VMAS环境适配

### 🤖 当前项目中的 VMAS 接入方式

当前工程通过 Hydra 组合配置来运行 VMAS 任务，入口是：

```bash
python train.py algorithm=recurrent task=vmas/flocking
```

关键文件：
- `train.py`：Hydra 训练入口
- `hydra_config.py`：将 Hydra 配置装配为 `Experiment`
- `experiment/experiment.py`：训练、恢复、渲染、结果展示主流程
- `conf/task/vmas/*.yaml`：任务参数
- `environments/vmas/*.py`：各任务的 `TaskConfig` dataclass

### 🧩 TaskConfig 与真实场景的关系

当前仓库中 `environments/vmas/*.py` 主要负责定义 Hydra 需要的 `TaskConfig`，例如：

```python
@dataclass
class TaskConfig:
    max_steps: int = MISSING
    n_agents: int = MISSING
    ...
```

这些文件本身**不是完整的场景逻辑实现**。实际环境仍由：

```python
env = make_env(
    scenario=self.scenario_name,
    num_envs=num_envs,
    device=self.config.device,
    continuous_actions=self.config.continuous_actions,
    wrapper=None,
    **env_kwargs,
)
```

也就是说：
- 本地 `TaskConfig` 负责配置结构和类型约束
- 真实场景逻辑来自 `vmas.make_env(scenario=...)`

### 📊 当前实现中的观测、动作与奖励格式

当前项目对 VMAS 的使用方式，和“把 agent 维直接拼进 batch”的写法不同。

在 `GenomeEvaluator` 中：

```python
obs = env.reset()
```

`obs` 的实际结构是：
- `obs` 是一个长度为 `n_agents` 的列表
- `obs[i]` 表示第 `i` 个智能体在所有并行环境中的观测
- `obs[i].shape == (num_envs, obs_dim)`

执行一步后：

```python
obs, rews, dones, info = env.step(actions)
```

其中：
- `actions` 是长度为 `n_agents` 的列表
- `actions[i].shape == (num_envs, action_dim)`
- `rews` 也是长度为 `n_agents` 的列表
- `rews[i].shape == (num_envs,)`

### 🔄 当前实现中的参数共享方式

当前项目使用**单个共享网络**处理所有智能体，但处理方式是：
- 先拿到 `obs[0]`，计算第 0 个智能体在所有环境中的动作
- 再拿到 `obs[1]`，计算第 1 个智能体在所有环境中的动作
- 依次循环所有 agent

也就是说，当前实现是“**按 agent 顺序循环调用同一个网络**”，而不是把 `(num_envs, n_agents, obs_dim)` reshape 成一个大二维张量后一次送入网络。

因此在当前工程里：
- `RecurrentNet` 的 `batch_size` 对应的是 `num_envs`
- 训练时通常使用 `experiment.trials`
- 不是 `num_envs * n_agents`

### 🏋️ 当前训练与评估流程

当前 `Experiment` 中与 VMAS 相关的实际流程如下：

1. Hydra 组合 `experiment / algorithm / task` 三组配置。
2. `Experiment._make_env()` 用任务配置创建 VMAS 环境。
3. `Experiment._make_net()` 当前只会创建 `RecurrentNet`。
4. `GenomeEvaluator` 或 `BatchGenomeEvaluator` 评估基因组。
5. 奖励按 agent 求平均，作为每个环境的全局奖励。
6. 再对所有并行环境的累计回报做统计。
7. `stats.mean` 被写入 `genome.fitness`，并进入 NEAT 选择与繁殖。

奖励聚合的实际逻辑是：

```python
rewards = torch.stack(rews, dim=1)   # (n_envs, n_agents)
global_reward = rewards.mean(dim=1)  # (n_envs,)
env_rewards += global_reward
```

所以当前项目优化的是：
- 每个环境内，所有智能体平均后的全局回报
- 再对所有并行环境取统计量

### ⚡ GPU 批量评估

当设备是 CUDA 且 `experiment.n_parallel > 1` 时，当前工程不会走多进程 CPU 评估，而是使用：

- `experiment/batch_evaluator.py`
- `pytorch_neat/batched_recurrent_net.py`

这条路径会：
- 一次打包多个基因组
- 使用 `BatchedRecurrentNet` 做批量前向传播
- 继续沿用当前项目的动作映射和全局奖励定义

这也是当前工程里最重要的性能优化之一。

### 🎯 当前项目对网络类型的实际支持

从当前 `train.py` 工作流看：

- `RecurrentNet`：已完整接入 Hydra 训练框架
- `AdaptiveLinearNet`：保留在 `pytorch_neat` 子库中，但未接入当前训练入口
- `AdaptiveNet`：保留在 `pytorch_neat` 子库中，但未接入当前训练入口

## 总结

### 🎯 关键要点

1. **训练流程**：
   - NEAT进化基因组
   - 基因组转换为网络（直接或通过CPPN）
   - 网络在环境中评估
   - 选择、交叉、变异

2. **CPPN**：
   - 是一个生成权重的神经网络
   - 输入：坐标、激活值、当前权重
   - 输出：初始权重或权重更新量
   - 利用空间规律性

3. **坐标设计**：
   - 反映语义关系
   - 利用对称性
   - 使用合理的范围
   - 按类型分层

4. **VMAS适配**：
   - 当前工程直接接入的是 `RecurrentNet`
   - 智能体之间参数共享，但按 agent 顺序循环前向
   - `obs` / `actions` / `rews` 都是按 agent 列表组织
   - 训练与分析围绕 Hydra、`Experiment`、benchmark 脚本展开
